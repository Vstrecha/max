# Admin Service

Django admin панель для управления системой мероприятий.

## Описание

Admin сервис предоставляет веб-интерфейс для администраторов системы. Позволяет управлять пользователями, мероприятиями, просматривать статистику и аналитику.

## Технологии

- **Django 5** - веб-фреймворк для Python
- **PostgreSQL** - база данных
- **Django Admin** - встроенная админ-панель

## Функциональность

### Управление пользователями (Profiles)
- Просмотр списка всех пользователей
- Детальная информация о пользователе
- История регистраций на мероприятия
- Статистика посещений

### Управление мероприятиями (Events)
- Просмотр всех мероприятий
- Статистика регистраций
- Статистика посещений
- Процент посещаемости
- Список участников с отметкой о посещении

### Управление участиями (Event Participations)
- Просмотр всех регистраций
- Отметка о посещении мероприятия

### QR Сканирования (QR Scans)
- История всех сканирований QR-кодов
- Информация о том, кто и когда сканировал

## Запуск

### Через Docker Compose

```bash
docker-compose up admin
```

### Локально

```bash
cd admin
pip install -r requirements.txt
python manage.py runserver 0.0.0.0:8001
```

## Создание суперпользователя

### Через Docker

```bash
docker-compose exec admin python create_superuser.py
```

### Вручную

```bash
docker-compose exec admin python manage.py createsuperuser
```

## Доступ

После запуска admin панель доступна по адресу:
- **Локально**: `http://localhost:8001/admin/`
- **Через nginx**: `https://your-domain.com/admin/`

## Переменные окружения

Admin использует те же переменные окружения, что и backend:
- `DB_HOST` - хост базы данных
- `DB_NAME` - имя базы данных
- `DB_USER` - пользователь базы данных
- `DB_PASSWORD` - пароль базы данных

## Миграции

Миграции применяются автоматически при первом запуске.

Для ручного применения:

```bash
docker-compose exec admin python manage.py migrate
```
